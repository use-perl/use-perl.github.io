Kwiki Hackery

gav on 2003-08-09T01:55:29

Risking the wrath of the code gods I did some evil things to CGI::Kwiki so that there could be an inc/ directory under database/ that would contain a custom header for pages.

The reason for the madness is that on my CPAN modules Wiki there's now a header for each page that matches a CPAN module. There's a cron job that runs a couple of times a day to keep the module list up to date from 02packages.details.txt and another than runs every 20 mins to keep the inc/ directory updated.

Update: Now also links to Acme's testing data for each module.
