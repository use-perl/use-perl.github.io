But, why are there so few Perl jobs out there?

Here's a stat that might illuminate the point: Sr. Software Engineer finds 103 hits on jobs.boston.com. Sr. software engineer perl returns 3. (Java and C++ return 12 and 11 respectively). Of those Perl positions, two expect Java / C++ as well. The third is actually a technical writer position.

Perhaps the lesson is that you have to have solid experience in multiple languages to be marketable today. Perl isn't a bad language to have, as long as you have Java or C++ as well. (For example, 48 positions on jobs.boston.com mention Perl in some context -- but compare that with 130 that mention Java and 89 that mention C++.)
