When a search is not a search

lilstevey on 2003-12-03T10:46:53

I've been looking through a few cpan modules, and I've found a few which mention 'search' in their title, and whilst I really don't want to be pedantic ( a cue to be pedantic if ever there was one ), I don't think a few of them are true searches, but are merely query facilitators.

I'm not going to mention any names, because I think that would be unfair, as I'm trying to make a general point, rather than criticise a volunteer that has made a lot of effort to give something back to the community, but in my mind, anything that maps a query directly onto an underlying column in a table ( "Artist like '%kylie%'" ) for example, is a query facilitator, whereas a search would involve building some kind of index on the data, and maintaining it with sensible caching in place.
