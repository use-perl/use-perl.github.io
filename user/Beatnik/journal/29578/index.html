DBD::Sybase virtual databases

Beatnik on 2006-05-10T13:09:00

One of the products we use, uses a backend database based on Sybase. The product offers a fallback mechanism.. Basically, you connect to a virtual database and the product decides weither to connect to the primary or secundary database. Sybase itself doesn't do this.. (Or rather, I haven't looked into Sybase but DBD::Sybase doesn't). It now seems that if you use the patched DBD::Sybase libraries as provided on the vendor site, it works.. It's still on my TODO list.
