Attribute Handlers Hurt

cwest on 2003-03-25T02:14:42

I want to be able to use Attribute Handlers in my poest plugins. Most importantly, :fork. Of course, this one requires a smart dispatcher of some sort, which requires another layer of abstraction. I don't want another layer of abstraction, however. I'm very confused right now because I don't know how to tell a method to fork to POE::Wheel::Run short of defining a fork event which will DTRT with a code reference. Now that I think about it, that's not an insane idea either, just not as down right pretty.

It's nice when you think solving blocking issues will be a walk in the park. The problem is that it's hardly a walk in the park for the implementor, me.
