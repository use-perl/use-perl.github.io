What if you look at a program and it really makes no sense at all?

Peter J. Scott, "Perl Medic, Transforming Legacy Code". http://www.perlmedic.com/

While I've got the chance I should give a review of this book. I've had my copy for probably a solid month now and I've read it twice. I've been at this programming thing for a long while now, largely using Perl. Peter's book has made my code better instantly.

Perl Medic brings together all the disparate bits of information and "it hurts when I do that" I've accumulated over the years, bringing them all within reach again.

I read this book twice so it would sink in better. Not only is Perl Medic an excellent reference, it's an excellent teacher.
