Perl 6 Test Suite Refactor - file and directory naming

particle on 2008-01-11T17:47:06

after conversations with larry and patrick, including a mailing list thread linked below, we've decided on a strategy for file and directory naming for the official Perl 6 specification tests currently stored in the pugs repository.

the directory structure looks like this:

t/spec/ S02-literals/ radix.t S02-whitespace/ comments.t S03-operators/ autoincrement.t comparison.t equality.t relational.t ... S29-functions/ str/ substr.t

please note the following:

~ the root directory for all specification tests is 't/spec/' ~ the top level directories under the root are named after the synopsis number and the section of the synopsis to which the contained test files correspond ~ top level directories may contain subdirectories ~ test file names reflect the subject being tested

i'll follow up soon with information about migrating tests to this new structure, and about new compiler directives allowing these tests to be used across Perl 6 implementations.
