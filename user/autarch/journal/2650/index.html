Then I pounded my head against a bug in mod_proxy with Apache 1.3.23 last night. It turns out that mod_proxy will drop all but the last one when dealing with multiple Set-Cookie headers. Considering most mod_perl sites use a dual-Apache setup with mod_proxy on the frontend that could affect a lot of people. Once I figure out the problem it was easy to simply go back to 1.3.22 but it took quite a while to figure out WTF was going on. No fun at all.
