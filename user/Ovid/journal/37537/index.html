Corsica!

Ovid on 2008-09-25T09:13:10

Tomorrow I leave for a week and a half in Corsica (any Corsican Perl geeks out there?). I'll likely be offline for much of that time, so this is a lousy time for me to ask for contributors to App::Prove::History :)

App::Prove::History is a preliminary project to save test results to a database. The code is a bit sloppy in places and the docs are, uh, lacking, but the basic usage as explained in the README works. Currently the only interface to the saved results is via the SQLite executable, but we'll work on that. It's very important that a proper API for test results be presented so that we're free to evolve the schema.

Yeah, you have to use git for it, but you wanted to learn that anyway, right? You can read the github guides to get up to speed. It's important to remember that even though the initial learning curve might seem daunting to some, it's actually very easy to use once you get it up and running.

While I'll be gone, I can't help you, but if anyone is willing to contribute, I'll give you commit access to the repository. I don't know git or github very well, but I'm pretty sure it's just a matter of my adding your public key, though I'm not sure how to do that yet.

enjoy!
