Matts on 2006-11-25T21:41:07

YAML came out as a reaction to XML, and for a brief while it was fine and easy for data serialisation, but then the YAML spec came out, and as the spec was refined -- the authors discovered that the issues that XML solves are actually HARD, and the spec grew and grew. The YAML spec is now about 5 times the length of the XML spec, and it is *way* harder to implement.

I'd stick with XML if I were you, but YMMV.
