CPAN version number retrograde

merlyn on 2003-01-16T21:28:04

Gah. Twice today, I've had to deal with CPAN version retrograde, where a PAUSE member uploads a new distro that has most module version numbers higher than existing distros, but one or more version numbers that are lower or even absent. This gives CPAN.pm the fits, and rightly so. CPAN.pm wants you to install the newer distro to upgrade most things, but then the older distro to upgrade the remaining few items, and the cycle repeats indefinitely.

I wish either that PAUSE would reject such distros, or that the PAUSE admins would give timely notification to the authors so that I wouldn't have to do this manually every time. It'd especially be nice if there was a FAQ page that I could simply point people to.

update: No, make that Three now.

Psh, XML::Filter::Dispatcher, and WWW::Search::Scraper. Gah.
