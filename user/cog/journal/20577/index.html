My Opinion

phillup on 2004-08-26T16:10:43

I've got a lot of the Perl books on your list, but I don't know where you are in needed / desired skills. I do mostly CGI programming with Mason... so, take this with that in mind.

If I was limited to one Perl book from the list, it would be Perl Cookbook. It is a great book to have around when you have a common problem and you want ideas for a starting point. Without knowing more about your specific needs or desires, I'd say that is one of the best generalized books on the list. I also found it useful when I knew ways to solve the problem in another language and wanted to know the "Perl way". (I've been accused of using Perl to write C++ code.;-))

I'd also say that while Programming the Perl DBI: Database Programming with Perl is a great book, it is dated in the extreme. If your database needs fall in the MySql category, then this book has a pretty good section on using DBI.

Perl and LWP is another book worth having if you absolutely must use that technology. But, WWW::Mechanize has made that much less necessary.

If either of the above two books were updated, I would go out and buy them immediately. But, I would put the current versions near the end of the list.

My most recent purchase has been Perl Medic and I really like the book. It has the best section on testing I've seen so far. I'm not sure if that says something good about the book, or something poor about how little information is in print on the subject... one thing that does bug me about this book is the way they typset the book. New lines of code are presented in italics (ever seen an italicized pipe... looks a lot like a slash) and it does not stand out well.

OTOH, Object Oriented Perl is a most excellent read. Not just for the subject matter, but because it is just plain easy to pick out changed lines of code due to the way they typeset it. Highly recommended if you are looking for another book on OO programming with Perl.

HTH
