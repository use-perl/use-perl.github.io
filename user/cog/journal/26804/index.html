Development machines by itself are not always good

cog on 2005-09-20T15:32:01

You might think that moving all your development to a development machine is a good idea.

Before doing it, please consider a few points:

* Make sure the development machine is enough for all the projects involved.

* Make sure you have policies regarding restarting the machine and so on.

The migration I'm doing should take about 3 hours to complete... Instead, it's been going for 6 hours, it's at about 90% now and sometimes I have to act and do something about some records not getting into the new database.

Oh, and I've just been told that there's the possibility that somebody else restarts the machine...

Oh well... at least I finished the migration scripts in time, which is what matters right now.
