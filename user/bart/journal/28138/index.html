Freedb sucks

bart on 2005-12-27T14:03:08

I was dubious about using this ttile, but there are at least two recent journals which use a title like "X sucks" so I went ahead and decided to do it too. Tell 'em how you really feel.

Well: Freedb (CDDB) is in desperate need of an overhaul. I've done several submissions to the CD database, and all of them have been rejected. No human intervention involved, it's all automatic.

What happened? Well, freedb has been designed with no clashes in mind. Let me tell you: entry clashes do occur, and it would appeaer that it happens even a lot more with these 2-track CDs AKA CD singles.

When using FreeDB/CDDB to look up a CD, you often manually have to select between several entries. So clashes do occur in the database... But not within one genre.

It looks like its architecture is first a split up in about 10 genres (rock, data, misc) — I think it's safe to say that these subgenres are actually independent databases. So you can have clashes between these databases, but not within one database.

Clashes do occur, even within a genre. Get your head out of the sand, and accept it.
