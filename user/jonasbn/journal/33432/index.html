Workflow moving from CVS to SVN

jonasbn on 2007-06-05T09:36:14

I have just completed the process of migrating the CPAN module 'Workflow' from CVS to Subversion. The project is hosted on SourceForge in order to open up for easier community development and open source activities.

SourceForge offers both CVS and SVN. We (the developers) decided however to move to Subversion.

I have had some problems getting cvs2svn to work on my MacBook Pro, but this here tip helped me out and I have converted the CVS repository to a dump, which has been imported into subversion.

Until now everything has worked out fine and the documentation from SourceForge is actually quite good. I would love it to include a different layout though and perhaps a step by step guide, instead of just a collection of prose.

The CVS repository will be disabled after I have evaluated that everything works as expected.

At the same time I have written up the road map for Workflow as a HTML document. This still requires some work, but I am attempting to get all the things in place so we can pick up on the development.
