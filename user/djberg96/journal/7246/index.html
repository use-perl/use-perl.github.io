Spreadsheet::WriteExcel test

jmcnamara on 2002-08-23T08:43:22

In the CPAN distribution of Spreadsheet::WriteExcel there is only has one real test and that is to determine if the hardware supports a 64 bit IEEE float.

The reason that there aren't more tests is that there isn't any way to test the module apart from creating a file, opening it in Excel, checking if the required features are okay and then closing it again. This isn't something that can be automated on a cross-platform basis.

It would be possible to generate an Excel file, MD5 the results and check it against a known value. However, this would be artificial.

This is not to say that the module hasn't been tested. I spent over 3 months on the development and testing of OLEwriter.pm alone. This involved creating binary files of various N, N-1, N+1 sizes, storing them with OLEwriter and then reading them with 3 versions of Excel on Windows, MS Works on Windows and OpenOffice, Gnumeric and Xess on Linux.

For each feature that is added to Spreadsheet::WriteExcel I have to go through a similar process. This, and the documentation, is what makes progress so slow. But again, it would be difficult to include these type of tests in the CPAN distribution.

In a way the 40+ example files are the tests. They serve as unit tests and as regression tests. And Excel is the arbiter of whether the module works or not.

The best way to test your ported version is to compare the binary output with the output of Spreadsheet::WriteExcel and ensure that there is byte-for-byte equivalence. In that way you will inherit my bugs but also my testing. :-)

John.
