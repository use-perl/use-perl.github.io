Write tests in different ways

brian_d_foy on 2005-07-04T19:17:01

At WWDC (yeah, that was a month ag and I still have it in a sticky note), someone was talking about testing (I think it was a Cocoa talk), and the speaker whom I failed to note, said something like:

Write test scripts in different ways because users will write them in different ways.


We all have our own particular styles and ways of coding, and that shows up in our test code too. However, subtle bugs might be poked if we used one of Perl's other ways to do it.

I think it's something I should probably think about, and it's in line right behind all the other things I should think about, and now it's in use.perl which means I know I should think about it, I want to get it off my desktop, and I'll never see it again.
