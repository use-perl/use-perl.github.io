Pod::Simple::HTML is (slightly) better

osfameron on 2006-07-11T15:58:53

I gave up using Pod::HTML for some automated pod creation because a) the documentation was unclear on which of the many many parameters needed to be set in order to link one pod to another, and b) links to modules with "::" would work, but modules like 'Template' or 'DBI' didn't, c) I hate the "the Foo::Bar manpage" link text.

Pod::Simple::HTML is very slightly better, though the code is still pretty hairy.

Try Pod::POM::View::HTML

jj on 2006-07-11T18:08:57

Try Pod::POM::View::HTML, that's one of the modules I use to create perldoc.perl.org. It's very useful because all the methods can be overrided if they don't output quite the HTML you want.
