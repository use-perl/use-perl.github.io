Leaving SourceForge already

jouke on 2003-08-14T15:50:36

I had the best intentions when I re-joined SourceForge to do the pVoice development there, but it didn't turn out to be what I expected.
I thought it would be a good place to store my CVS stuff. However, the ViewCVS there is broken, the communication with the SF cvs server is slow (terribly slow) from here, I probably won't use the File Release system, I don't need a place for my homepage (thanks to the ByteMark guys), I already have mailinglists (for two years), so what do I need SF.net for then?

So I picked up my CVS repository, put it on my new ByteMark host, installed ViewCVS and CVSMonitor and look what I've got:

My Own ViewCVS for pVoice related things
and My Own MonitorCVS for pVoice related stuff

Now on my todo list:

  • Updated website
  • Seperate 'developers' section on the website (in case anyone wants to join)
  • Writing the helpfile for pVoice 2.1
  • Releasing pVoice 2.1
Anyway: the new server is up and running. Webserver works, mail works, mailinglists work, even CVS works there....

I'm one happy fellow
