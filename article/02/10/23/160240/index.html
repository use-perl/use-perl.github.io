autarch on 2002-10-23T20:43:24

From the masonbook.com [masonbook.com] website, the entire contents will be freely available online.

I should point out that this might take a little while to happen, because there are some challenges ahead of us in doing this.

For the curious, here's the deal. The book was written in a mostly-POD format ORA calls "pseudo-POD". It's basically POD with a few extras like R<>, which is used to indicate text that is "replaceable" (like a host name), or U<>, for URLs.

Then ORA converts this to DocBook XML internally, and again to FrameMaker for publication. At some point during this stage, the editors made a number of changes to make the book not suck. But those changes were not made in the source format (pseudo-POD).

So we need to somehow integrate those changes. That's challenge #1. ORA tells us they should be able to convert from FrameMaker back to pseudo-POD, so it shouldn't be _too_ hard to do.

Then we have to convert the pseudo-POD into HTML ourselves. Thankfully, Sean Burke finally released his Pod::Simple modules so that POD parsing no longer has to suck. And his design makes it easy to handle the pseudo-pod extras. Yay Sean!

But it'll still take some serious coding to do the conversion. I figure it'll be online sometime in December or January, at best.

The nice thing about being able to distribute it online is that we'll be able to release updated versions reasonably often, once we have our tools in place. And of course if people are really motivated, they could send proposed changes as diffs, cause we'll make the POD source available too.

I do hope ORA puts it in Safari though.
